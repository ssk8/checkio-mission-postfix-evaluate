Wenn arithmetische Ausdrücke in der vertrauten Infix-Notation 2 + 3 × 4 angegeben werden, weichen die Klammern in der Auswertungsreihenfolge von der üblichen PEMDAS ab, die durch den Vorrang und die Assoziativität der einzelnen Operatoren bestimmt wird. Die alternative Postfix-Notation (auch bekannt als Reverse Polish Notation) mag für Menschen, die an die herkömmliche Infix-Notation gewöhnt sind, zunächst seltsam erscheinen. Es stellt sich jedoch heraus, dass die Postfix-Notation für Maschinen einfacher ist, da sie die Kodierung jeder beabsichtigten Auswertungsreihenfolge ohne Klammern ermöglicht!

Um einen Postfix-Ausdruck auszuwerten, verwenden Sie einen zunächst leeren Stapel. Durchlaufen Sie die Elemente eines nach dem anderen von links nach rechts. Wenn das aktuelle Element eine ganze Zahl ist, fügen Sie es einfach an das Ende der Liste an. Andernfalls ziehen Sie zwei Elemente vom Ende der Liste ab, um diese Operation durchzuführen, und fügen das Ergebnis wieder an die Liste an. Unter der Annahme, dass items ein zulässiger Postfix-Ausdruck ist, was bei diesem Problem garantiert ist, so dass Sie keine Fehlererkennung oder -behebung durchführen müssen, wird die einzige Zahl, die sich noch auf dem Stapel befindet, zur endgültigen Antwort, sobald alle Elemente verarbeitet wurden.
